# Git vs GitHub

## What is Git?

Git is a Distributed Version Control System (DVCS) that helps developers track project history and manage changes locally.

---

## What is GitHub?

GitHub is a cloud-based platform that hosts Git repositories online. It allows developers to collaborate, share code, and maintain a remote backup of their projects.

---

## Git vs GitHub

| Git | GitHub |
|------|---------|
| Version Control System | Cloud-based Hosting Platform |
| Installed on the local machine | Accessible through the internet |
| Manages project history | Stores remote repositories |
| Works without internet for most operations | Requires internet access |
| Creates commits | Stores commits after `git push` |

---

## Similarities

- Both are used together in software development.
- Both help developers manage projects efficiently.
- Git and GitHub support collaboration and version control workflows.

---

## Real-Life Analogy

Think of writing a document.

- **Git** is like **Microsoft Word**, where you create and edit your document.
- **GitHub** is like **Google Drive**, where you store, share, and collaborate on that document.

---

## Key Takeaways

- Git and GitHub are different tools.
- Git manages the project locally.
- GitHub stores a remote copy of the project.
- Git can work without GitHub.
- GitHub depends on Git repositories.

---

## Common Interview Mistakes

❌ Git and GitHub are the same.

✔ Git is a Version Control System.

✔ GitHub is a cloud platform that hosts Git repositories.

---

## My Understanding

I learned that Git and GitHub are not the same. Git is responsible for tracking project history and managing changes locally, while GitHub stores the remote repository, making collaboration, sharing, and project backup much easier.
