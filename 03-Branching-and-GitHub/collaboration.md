# GitHub Collaboration

## Overview

GitHub provides collaboration features that allow developers to contribute to projects, review code, report issues, and work together efficiently.

---

## Fork

A Fork creates a personal copy of another developer's repository under your GitHub account.

It allows you to make changes without affecting the original project.

---

## Pull Request (PR)

A Pull Request is a request to merge your changes into another branch or repository.

Before merging, maintainers can review, discuss, and approve the proposed changes.

---

## Issues

GitHub Issues are used to report bugs, request new features, ask questions, or discuss improvements related to a project.

They help teams organize and track development tasks.

---

## Collaboration Workflow

```text
Original Repository
        │
        ▼
Fork Repository
        │
        ▼
Clone Repository
        │
        ▼
Create Branch
        │
        ▼
Make Changes
        │
        ▼
Commit Changes
        │
        ▼
Push Changes
        │
        ▼
Create Pull Request
        │
        ▼
Code Review
        │
        ▼
Merge Changes
```

---

## Key Takeaways

- Fork creates your own copy of a repository.
- Pull Requests are used to propose changes.
- GitHub Issues help track bugs and feature requests.
- Collaboration becomes easier with GitHub's workflow.
